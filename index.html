    Wildcards

  • Buscar toda la información dentro de una carpeta: ls *
  • Buscar toda la información dentro de una carpeta especificamente: ls *.(tipo-de-archivo)
  • Buscar toda la información dentro de una carpeta con un nombre especificamente y varios caracteres despues: ls (nombre-del-archivo)*
  • Buscar toda la informacion dentro de una carpeta con un nombre especificamente con el numero caracteres que le asignemos: ls (nombre-del-archivo)?
  • Buscar y obtener toda la informacion de dirtectorios y archivos que empiencen con un letra mayuscula: ls -d [[:upper:]]*
  • Buscar y obtener toda la informacion de dirtectorios y archivos que empiencen con un letra miniscula: ls -d [[:lower:]]*
  • Buscar toda la informacion de archivos y directorios por caracteres asignados: ls [(letra-para-la-busqueda)]*

    Redirecciones: cómo funciona la shell

  • Redirige la salida de un comando a un archivo. El mismo sobrescribe el contenido del archivo a donde se redirige la salida: ls (nombre-del-archivo-o-diretorio) > (nombre-del-archivo-a-crear).txt
  • Redirige el input de un comando hacia un archivo.: ls (nombre-del-archivo-o-diretorio) < (nombre-del-archivo-a-crear).txt
  • Concatena la salida de un comando a un archivo, si no existe el archivo lo crea: ls (nombre-del-archivo-o-diretorio) >> (nombre-del-archivo).txt
  • Redirige la salida de error de un comando a un archivo: ls (nombre-del-archivo-o-diretorio) 2> (nombre-del-archivo).txt
  • Redirige la salida de un comando, que se ejecuto satisfactoriamente o un comando que presento errores, a un archivo: ls (nombre-del-archivo-o-diretorio) > (nombre-del-archivo) 2>&1

    Redirecciones: pipe operator

  • Genera un stdout con el texto que tenemos: echo (texto)
  • Muestra los dos archivos concatenados: cat (nombre-del-archivo) (nombre-del-archivo)
  • El pipe operator "|" hace que el stdout de un comando sea el stdin de otro comando: ls -lh | less (nombre-del-archivo)
  • "tee" hace algo parecido a "">"", pero dentro de los pipes: ls -lh | tee (nombre-del-archivo) | less
  • Generar texto con bonitos colores: echo "texto" | lolcat
  • Generar una vaquita con un mensaje de texto: cowsay "texto"
  • Generar una vaquita con un mensaje de texto con color, usando el pipe operator: cowsay "texto" | lolcat

    Cómo se manejan los permisos

    Cuando listamos con ls -l se muestran varias cosas. Los tipos de archivos:

  • - archivo normal.
  • d directorio.
  • l link simbólico.
  • b archivo de bloque especial.

    • Tipos de modos:

    rwx corresponde con read, write y execute. Se representan con 3 bits, y los podemos manejar a través de un modo octal, esto es, pasar de binario a número.

  • rwx (1,1,1) dueño. En modo octal es 7.
  • r-x (1,1,1) grupo. En modo octal es 5.
  • r-x (1,0,1) world. Octal 5.

    • Modo simbólico:

    Esto es para asignar los permisos a los diferentes posibles usuarios.

  • u Solo para el usuario.
  • g Solo para el grupo.
  • o Solo para otros (world).
  • a Aplica para todos.

    Modificando permisos en la terminal

  • Crear archivos: > (nombre-del-archivo)
  • Crear o editar informacion del archivo: cat > (nombre-del-archivo)
  • Cambiar los permisos de un archivo: chmod (numero-de-permiso-de-usuario)(numero-de-permiso-de-grupo)(numero-de-permiso-de-mundo) (nombre-del-archivo)
  • Quitar los permisos a alguien en particular: chmod (letra-de-alguien) - (letra-de-permiso-a-quitar) (nombre-del-archivo)
  • Poner los permisos a alguien en particular: chmod (letra-de-alguien) + (letra-de-permiso-para-asignar) (nombre-del-archivo)
  • Asignar varios permisos al mismo tiempo: chmod (letra-de-alguien) - (letra-de-permiso-a-quitar),(letra-de-alguien)=(letra-de-permiso-para-asignar) (nombre-del-archivo)
  • Saber que usuarios somos: whoami
  • Saber que usuarios somos con ID: id
  • Cambiar de usuario hacía root: su root
  • Otorga temporalmente los permisos de root para ejecutar algún comando que ocupe permisos especiales: sudo (comando)
  • Cambiar contraseña de algun usuario: passwd

    Comandos de búsqueda

  • Buscar en todas las rutas del PATH para encontrar donde está alojado algún archivo binario: which (nombre-del-programa)
  • Encontrar un archivo a partir de una ruta inicial, y dentro de todas las carpetas que surjan de ese inicio: find (ruta-inicial) -name (nombre-del-archivo-a-buscar)
  • Encontrar a partir de una ruta inicial por tipo de archivos o directorios: find (ruta-inicial) -type ("d" directorios, "f" documentos) -name (nombre-del-archivo-a-buscar)
  • Encontrar a partir de una ruta inicial archivos por tamaño: find (ruta-inicial) -size (tamaño)(unidad)
  • Encontrar a partir de una ruta inicial archivos o directorios limitando la busqueda: find (ruta-inicial) -type ("d" directorios, "f" documentos) -maxdepth 2
  • Encontrar a partir de una ruta inicial archivos o directorios vacios: find (ruta-inicial) -type ("d" directorios, "f" documentos) -empty

    Usando el comando grep

  • Encontrar coincidencias en un archivo: grep (palabra) (nombre-del-archivo)
  • Encontrar coincidencias en un archivo ignorando si es mayúscula o minúscula: grep -i (palabra) (nombre-del-archivo)
  • Encontrar ocurrencias y contar el numero de ocurrencias que hay en un archivo: grep -c (palabra) (nombre-del-archivo)
  • Encontrar ocurrencias las cuáles son los resultados que NO coinciden con tu expresión regular: grep -v (palabra) (nombre-del-archivo)
  • Encontrar ocurrencias pero limitando la busqueda en líneas: grep -m (numero-de-ocurrencias) (palabra) (nombre-del-archivo)
  • Contar la cantidad de palabras, lineas y bits que hay en un archivo: wc (nombre-del-archivo)
  • Contar el numero de lineas que hay en un archivo: wc -l (nombre-del-archivo)
  • Contar el numero de palabras que hay en un archivo: wc -w (nombre-del-archivo)
  • Contar el numero de bits que hay en un archivo: wc -c (nombre-del-archivo)

    Comprimiendo archivos tar y zip

  • .tar se usa mucho en repositorios. Para comprimir archivos: tar -cvf (nombre-del-archivo).tar (nombre-de-la-carpeta-a-comprimir) donde c → compress, v → verbose, f→ file.
  • .gz es un poco mejor, el mismo comando pero con la bandera z→zip: tar -cvzf (nombre-del-archivo).gz (nombre-de-la-carpeta-a-comprimir) El algoritmo gzip es muy eficiente para comprimir.
  • Para descomprimir es el mismo comando pero con la bandera x → decompress en lugar de c → compress. Para que funcione, debemos descomprimir usando el mismo tipo de compresión (tar o zip)
  • .zip es uno muy común: zip -r (nombre-del-archivo).zip (nombre-del-directorio) Se comprime pero de forma recursiva con -r
  • Para descomprimir en zip usamos: unzip (nombre-del-archivo).zip
  • Esta tambien el rar se usa igual que el zip, para comprmir es: rar -r (nombre-del-archivo).rar (nombre-del-directorio) Se comprime pero de forma recursiva pero con -r
  • Para descomprimir en rar usamos: unrar (nombre-del-archivo).rar

    Editores de texto en la terminal

    Una de las utilidades más importantes de la terminal es el editor de texto. Hay diferentes opciones, pero Vim es uno de los mas sencillos y populares.

  • vi es la versión vieja de editor de texto: vi (nombre-del-archivo)
  • vim es la version moderna o pues (Actualizada): vim (nombre-del-archivo)
  • Al igual que con less para buscar una palabra, podemos hacerlo en Vim con: /(palabra) Te lleva a la primera coincidencia.
  • Para eliminar una línea, desde el modo normal (si estamos en modo INSERT solo presionamos Esc y listo), nos ponemos al inicio de la línea y presionamos la tecla d dos veces.
  • Para guardar nos ubicamos en el modo NORMAL, presionamos / y escribimos la letra w, Seria asi: /w
  • Para guardar y salir del editor de texto nos ubicamos en el modo NORMAL, presionamos / y escribimos las letras wq
  • Para salir de forma forzada por alguna razon que no te deje seria con ! ejemplo: /wq!
